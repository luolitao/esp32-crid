# 2026-06-03

## 修复 sniffer 漏掉 Message Counter 字节导致解析失败

- **根因**：`crid_sniffer.c` 在提取 Vendor Specific IE 数据时只跳过了 `ID + Len + OUI(3) + OUI_Type(1) = 6` 字节，但发送端（`main_tx/crid_messages.c`）的格式是 `OUI(3) + VendorType(1) + **MessageCounter(1)** + PackedMsg`，即 OUI Type 之后还有一个 Counter 字节。
- **修复**：
  - `data_offset`：`offset + 6` → `offset + 7`
  - `vendor_data_len`：`ie_length - 4` → `ie_length - 5`
  - 同时更新了非 RID Vendor IE 采样路径和 `crid_parser.c` 的注释
- **影响**：之前 data[0] 拿到的是 Counter 字节（0-255），被当作 Message Pack header 解析自然失败；现在 data[0] 正确指向 Packed Message header (0xF2) 或单消息头。
